Technical Architecture

The simulation is composed of six Python BehaviorScripts, each attached to a USD prim. They communicate exclusively through USD attributes and a carb singleton registry — no direct imports between scripts, no hard-coded dependencies.

Core Design Principles

01

USD Stage is the Data Bus

Every piece of shared state — traffic light phase, road curves, spawn config — lives as a USD attribute on a prim. Any script can publish or subscribe at any time.

02

The Scene is the Config

Configure every module entirely through prim properties in the Omniverse UI. No code changes needed to add a new intersection, traffic light, or vehicle type.

03

Scripts are Decoupled

VehicleController doesn't call Vehicle directly — it spawns a prim and attaches the script. TrafficLight doesn't know Vehicle exists — it just writes currentState.

MODULES

Script Reference

🟢
MANAGER / SPAWNER

VehicleController.py

Owns the full lifecycle of all cars in the scene. Never drives — that's Vehicle's job.

  • Loads Car Pool dataset (USD / JSON asset list)
  • Dynamically spawns car prims into USD Stage
  • Assigns Road Network curves to each spawned car
  • Attaches Vehicle.py BehaviorScript to each prim
  • Staggered startDelay prevents spawn overlap
  • Recycles & despawns cars that complete routes
🚗
PER-CAR BEHAVIOUR

Vehicle.py

Auto-attached by VehicleController. Handles everything a single car does.

  • BasisCurves spline following + route chaining
  • Look-ahead orientation · per-seed random speed
  • Signal stop FSM:
    FREE→ WAITING→ CROSSED
  • All-stop intersection queuing:
    FREE→ WAIT→ PROC→ CROSS
  • Three-zone gap keeping · follow target mode
🚦
BEHAVIORSCRIPT

TrafficLight.py

  • GREEN → YELLOW → RED → GREEN cycle
  • Individually configurable phase durations
  • Phase offset + startState for staggering
  • Bulb albedo via shader diffuse_color_constant
  • Writes currentState Token to USD on each transition
🛑
BEHAVIORSCRIPT

AllStopIntersection.py

  • FIFO arrival-order queue
  • Right-of-way by arrival time
  • stopDuration enforcement
  • clearanceDelay between departures
  • register_arrival · can_proceed · clear
  • Instance in carb._intersection_registry
🎥
BEHAVIORSCRIPT

FreeCamera.py

  • Manual: WASD + QE navigation, Shift 3× speed
  • Follow: set followTargetPath to any car prim
  • Offset along car's local X axis (offsetX)
  • Height in world Y (offsetY)
  • Pitch + yaw from look-direction vector
  • followSmoothing lerp prevents jitter

Inter-Module Communication Channels

USD ATTRIBUTE BUS

Primary communication channel. currentState, transforms, config floats — all scripts read/write via USD prim attributes.

CARB SINGLETON REGISTRY

carb._intersection_registry — Vehicle scripts call AllStopIntersection's register_arrival, can_proceed, and clear methods.

MODULE-LEVEL _CAR_REGISTRY

Shared dict keyed per curve path. Enables per-curve gap detection between any two cars sharing the same road segment.

BEHAVIORSCRIPT LIFECYCLE

All scripts follow the same pattern:
on_init → on_play → on_update(dt) → on_stop
Config is read at on_play; on_update uses cached values only.

DEBUG

Troubleshooting

Containers not running on Brev

Run docker ps to check running containers. Expected: web-viewer, vscode, dev-nginx-1

docker compose down
docker compose up -d
Viewer tab won't connect

Keep only one viewer tab open at a time. Refresh the tab after restarting containers.

Cars not stopping at intersection on second lap

_intersection_state must reset to FREE at the loop boundary alongside _stop_state. Ensure you are on the latest Vehicle.py.

Camera snapping back when rotating

FreeCamera.py must not write rotation each frame. Older versions wrote a fixed Y=-90 which conflicts with Omniverse's built-in right-click mouse look.

Navigation speed too fast on Brev streaming

Run set_navigation_speed.py in the Script Editor and set ROTATION_SPEED = 0.3.
